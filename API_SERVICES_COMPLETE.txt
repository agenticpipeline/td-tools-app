OFFLINE SERVICES (src/services/offline/)
───────────────────────────────────────
11. cache.ts (393 lines)
    - Three services for offline support
    
    A. CacheService (TTL-based caching)
       Methods:
       * set<T>(key, data, ttl?) - Store with optional TTL
       * get<T>(key) - Retrieve if not expired
       * has(key) - Check if cached and valid
       * delete(key) - Remove single entry
       * clearMatching(pattern) - Pattern-based deletion
       * clear() - Clear all cache
       * getStats() - Get cache statistics
       
    B. OfflineQueueService (Operation queueing)
       Methods:
       * enqueue(operation) - Add to offline queue
       * getQueue() - Retrieve all pending operations
       * remove(id) - Remove after sync
       * incrementRetry(id) - Track retry attempts
       * clear() - Clear queue
       * getStats() - Get queue statistics
       
    C. NetworkStatusService (Network detection)
       Methods:
       * subscribe(listener) - Listen to network changes
       * getStatus() - Get current network status
       * isConnected() - Boolean check
       - Detects: wifi, cellular, none, unknown
       - Provides signal strength (0-100%)
       
    D. MemoryStorage (React Native fallback)
       - In-memory storage for React Native
       - Same API as localStorage
       - Used when localStorage unavailable

================================================================================
                        INTEGRATION POINTS
================================================================================

Express API (NUC):
  Client HTTP → apiClient (Axios) → http://localhost:3001/api/v1
                                     ├── GET /projects
                                     ├── POST /projects
                                     ├── GET /projects/:id/equipment
                                     ├── POST /projects/:id/equipment/parse
                                     ├── POST /projects/:id/estimate
                                     ├── POST /projects/:id/report/export
                                     └── ... (30+ endpoints)

React Native/Expo Components:
  Component → Hook (useState) → apiClient (async) → Response
                ↓
             State Update → Re-render

Storage Layer:
  Cache/Queue → localStorage (browser)
             → MemoryStorage (React Native fallback)
             → Auto-cleanup (TTL expiration)

Network Detection:
  App → NetworkStatusService → Network events
           ↓
     Online → Sync offline queue
     Offline → Queue operations
